# 工程实现逻辑

这篇文档用于同步当前工程代码进度。目标读者是没有上下文的编程模型或开发者。理论解释看 `docs/扰动图理论.md`，跑实验看 `docs/exp.md`。

## 1. 当前主线

项目已经从“让 VAE 生成 MILP 约束矩阵”调整为：

```text
RailDisruptLab 生成扰动数学图
VAE 在数学图上训练和生成 task output
RailDisruptLab 解码 task output，并继续 build / solve / analyze
```

工程边界：

```text
RailDisruptLab:
  负责铁路语义、锚点、规则、解码、回灌和 MILP 验证

backend:
  负责 Web/API、任务快照、Pueue 队列、项目数据流、场景表单构造和批量采样

VAE:
  只读取数值节点、数值边、task 规则和监督标签
  不读取车次名、站名、区间名、YAML、MILP 变量或约束矩阵
```

## 2. 关键文件

```text
core/vae_learning_graph.py
  scenario/config/disturbance_graph 与 VAE math graph 之间的转换
  定义 pool、edge type、task、target relation、decode 逻辑

VAE/src/data.py
  读取 vae_math_context_graph 和 vae_math_learning_sample
  转成 MathGraphSample / EdgeBatch / TargetData

VAE/src/model.py
  RailDisturbanceVAE
  typed message passing + context encoder + target encoder + VAE decoder

VAE/src/train.py
  训练入口，写 best_model.pt / last_model.pt / training_summary.json

VAE/src/generate.py
  从 context graph 生成 vae_math_generated_graph

VAE/src/evaluate.py
  graph/task output 相似性评估

scripts/project.py
  后端任务调度的低层调试入口
  管理 newproject / prepare / normal_generate / dataset / build / solve / export_timetable / model train / generation

backend/scenarios.py
  场景集合管理、单个场景表单写入、普通分布批量采样

scripts/decode_typed_generated_graph.py
  单个 generated graph 解码

scripts/decode_import_generated_graphs.py
  批量 generated graph -> disturbance graph -> scenario config

core/project_layout.py
  project 沙箱目录约定
```

旧 ACM-MILP 的 MILP 二分图转换、MIS/CA/SetCover 配置、Hydra 训练入口和旧 benchmark 流程已移除。

## 4. 数据接口

### 4.1 Context Graph

`context.json` 是共享上下文：

```json
{
  "schema_version": 1,
  "graph_type": "vae_math_context_graph",
  "decode_handle": {},
  "rules": {
    "pools": [],
    "edge_types": [],
    "tasks": [],
    "target_relation_feature_dim": 6
  },
  "graph": {
    "pool_x": {},
    "edges": []
  }
}
```

主要字段：

- `decode_handle`: VAE 不学习，只复制到生成结果，用于后续解码。
- `rules.pools`: pool id、pool size、feature dim。
- `rules.edge_types`: edge type id、source pool、target pool、feature dim。
- `rules.tasks`: task id、target pool、max slots、param dim、参数边界和约束。
- `graph.pool_x`: 每个 pool 的数值特征矩阵。
- `graph.edges`: pool index 之间的数值边。

当前 pool：

```text
pool 0 = event anchor, feature_dim = 6
pool 1 = section anchor, feature_dim = 4
```

当前 edge type：

```text
edge 0 = event-event, feature_dim = 3
edge 1 = section-section, feature_dim = 1
edge 2 = event-section, feature_dim = 2
```

### 4.2 Learning Sample

每个 `graph/samples/*.json` 是一个训练样本：

```json
{
  "schema_version": 1,
  "graph_type": "vae_math_learning_sample",
  "context_ref": "context.json",
  "supervision": {
    "targets": {},
    "target_relations": []
  }
}
```

`targets` 按 task 保存：

```text
task 0 = event delay
task 1 = section speed_limit / interruption
```

每个 task 的监督字段：

```text
count
anchor_index
params
```

`target_relations` 是训练辅助关系，字段包括：

```text
left_task_id
left_slot
right_task_id
right_slot
x
```

sample 不重复保存 `rules`、`pool_x` 和 `edges`。

### 4.3 Dataset Profile

`dataset_profile.json` 不是 VAE 输入，只用于解释和排查。

它保存：

- pool 行号到 anchor id 的映射。
- feature names。
- task、param、edge 的可读说明。
- source config 列表。
- export 参数。
- decode contract。
- debug 信息。

### 4.4 Generated Graph

VAE 输出：

```json
{
  "schema_version": 1,
  "graph_type": "vae_math_generated_graph",
  "decode_handle": {},
  "task_outputs": {
    "0": {
      "count": 1,
      "anchor_index": [15],
      "params": [[0.0069]]
    },
    "1": {
      "count": 1,
      "anchor_index": [3],
      "params": [[0.35, 0.04, 0.22857143]]
    }
  }
}
```

RailDisruptLab 读取 `decode_handle + task_outputs`，恢复标准 `disturbance_graph`，再回灌为 scenario config。

## 5. 训练图导出流程

训练图导出入口在 `scripts/project.py`：

```text
train_model(...)
  -> export_training_graphs(...)
  -> scenario_to_typed_vae_learning_graph(...)
  -> typed_learning_graph_to_math_context_graph(...)
  -> typed_learning_graph_to_math_learning_sample(...)
  -> infer_math_dataset_schema(...)
  -> write context.json / samples/*.json / dataset_profile.json
```

命令入口：

```bash
python scripts/project.py <project> model train <model_id> <scenario_set_id>
```

`export_training_graphs` 的核心逻辑：

```text
1. 从 CLI 参数接收 model_id、scenario_set_id 和训练参数
2. 根据 scenario_set_id 扫描 scenario_sets/<scenario_set>/*.yml
3. 每个 scenario YAML 解析为 ScenarioConfig
4. 调用 scenario_to_typed_vae_learning_graph
5. 第一个样本生成共享 context_graph
6. 后续样本必须生成完全相同的 context_graph
7. 每个样本写 graph/samples/<case>.json
8. 根据所有 samples 推断 count_bounds / param_bounds
9. 写 graph/math_context.json、graph/dataset_profile.json 和训练产物中的 training_config.json
```

图生成链路：

```text
prepare:
  source Excel
    -> projects/<project>/context.json
    -> BaseContext

model train:
  BaseContext + scenario YAML
    -> disturbance_graph
    -> typed_vae_learning_graph
    -> vae_math_context_graph
    -> vae_math_learning_sample
```

### 5.1 BaseContext 到 context pools

实现函数：

```text
_context_pools(base_context)
```

生成两类 pool：

```text
event anchors   -> pool 0
section anchors -> pool 1
```

同时返回：

```text
event_index:   anchor_id -> event pool row
section_index: anchor_id -> section pool row
```

这些 index 后续用于把真实扰动落点转成 `anchor_index`。

### 5.2 BaseContext 到 context edges

实现函数：

```text
_context_edges(...)
```

内部生成三类辅助边：

```text
_event_event_aux_edges(...)
_section_section_aux_edges(...)
_event_section_aux_edges(...)
```

筛边规则：

```text
event-event:
  same_train 或 same_station 或 planned_time_diff <= event_time_window
  每个 anchor 最多保留 event_top_k 个候选关系

section-section:
  shared_endpoint 或 adjacent_section 或 section_order_diff <= section_order_window

event-section:
  event_station_is_section_endpoint 或 train_route_contains_section
```

写入 math graph 时，边字段统一为：

```text
edge_type_id
source_pool_id
source_index
target_pool_id
target_index
directed
x
```

### 5.3 disturbance_graph 到 targets

实现函数：

```text
_targets_from_disturbance_graph(...)
```

转换规则：

```text
delay:
  disturbance role edge target = event anchor id
  event_index[anchor_id] -> anchor_index
  delay_seconds / DAY_SECONDS -> params[0]

speed_limit / interruption:
  disturbance role edge target = section anchor id
  section_index[anchor_id] -> anchor_index
  start_time / DAY_SECONDS -> params[0]
  duration / DAY_SECONDS -> params[1]
  speed_limit / 350 -> params[2]
```

target count 不能超过 `max_slots`。

### 5.4 disturbance_graph 到 target_relations

实现函数：

```text
derive_disturbance_relation_features(...)
```

处理方式：

```text
1. 先把每个 disturbance 映射成 task_id + slot + anchor/time/space 信息
2. 枚举两两扰动
3. 计算关系特征
4. 写成 target_relations
```

当前关系特征：

```text
same_anchor
spatial_near
time_overlap
time_gap_norm
delay_section_route_near
same_time_neighborhood
```

这些关系只用于训练 target encoder，生成阶段不输出。

### 5.5 schema 推断

实现函数：

```text
infer_math_dataset_schema(context_graph, learning_samples)
```

它会根据训练样本更新：

```text
rules.tasks[*].max_slots
rules.tasks[*].count_bounds
rules.tasks[*].param_bounds
rules.tasks[*].slot_constraints
```

注意：这里的 `max_slots` 最终会收敛到训练集中观察到的最大 count，而不是永远使用配置里的默认上限。

## 6. VAE Reader

训练 reader：

```text
RailDisturbanceDataset(graph_root)
  -> 读取 graph_root/context.json
  -> 扫描 graph_root/samples/*.json
  -> 每个样本组合成 MathGraphSample
```

生成 reader：

```text
RailDisturbanceContextDataset(graph_root)
  -> 读取 vae_math_context_graph
  -> 构造空 targets
  -> 复用 decoder 输出格式
```

内部统一结构：

```text
MathGraphSample:
  decode_handle
  pool_rules
  task_rules
  edge_type_rules
  pool_x
  edges
  targets
  target_relation_index
  target_relation_x
```

`EdgeBatch`：

```text
edge_index: [2, num_edges]
edge_attr:  [num_edges, feature_dim]
directed:   [num_edges]
```

`edge_index[0]` 是 source index，`edge_index[1]` 是 target index。

## 7. VAE Model

模型入口：`VAE/src/model.py`。

默认配置：

```text
hidden_dim = 64
latent_dim = 16
message_passing_steps = 2
```

Context encoder：

```text
event raw feature   -> pool encoder -> hidden_dim
section raw feature -> pool encoder -> hidden_dim
edge raw feature    -> edge encoder -> hidden_dim
typed message passing
pool mean + edge mean -> context
```

message passing：

```text
source_h + target_h + edge_h -> message
message 按 target index 聚合
old_node_h + agg -> update -> new_node_h
```

如果 `directed = false`，同时计算反向 message。

每类 edge 使用独立 message layer：

```text
event-event
section-section
event-section
```

Target encoder：

```text
count
anchor_index
params
target_relations
  -> target embedding
```

VAE 路径：

```text
训练:
  context -> prior
  context + target -> posterior
  posterior sample z + context -> decoder

生成:
  context -> prior
  prior sample z + context -> decoder
```

Decoder 对每个 task 输出：

```text
count_logits
anchor_logits
params
```

anchor 预测是分池的：

```text
task_id -> target_pool_id
slot query @ target_pool_embeddings.T -> anchor logits
```

因此 task 0 只在 event pool 选，task 1 只在 section pool 选。

Loss：

```text
count_loss  = cross entropy
anchor_loss = cross entropy，只算真实 count 范围内的 slot
param_loss  = smooth L1，只算真实 count 范围内的 slot
kl          = posterior 和 prior 的 KL
```

`max_slots` 只是张量上限，超过真实 `count` 的 slot 是 padding。

## 8. 主入口命令

以下命令仅作为低层调试入口；常规实验应在 Web 表单中完成。

初始化：

```bash
python scripts/project.py newproject <project>
```

准备共享 context：

```bash
python scripts/project.py <project> prepare --timetable-filename <file.xlsx> --mileage-filename <file.xlsx>
```

生成普通模拟 scenario set：

```bash
python scripts/project.py <project> scenario-set create <scenario_set>
python scripts/project.py <project> normal_generate <scenario_set>
```

构建 dataset：

```bash
python scripts/project.py <project> dataset create <dataset>
python scripts/project.py <project> build <scenario_set> <dataset>
```

求解和导出时刻表：

```bash
python scripts/project.py <project> solve <dataset>
python scripts/project.py <project> export_timetable <dataset>
```

训练：

```bash
python scripts/project.py <project> model train <model> <scenario_set>
```

生成并解码 scenario set：

```bash
python scripts/project.py <project> generation <model> best_model.pt <scenario_set>
```

完整验收命令以 `docs/exp.md` 为准。

## 9. 当前已完成

- project layout 已切到 `source/`、共享 `context.json`、`scenario_sets/`、`datasets/`、`model/`。
- FastAPI + Pueue + `backend.runner` 是当前平台主入口；`scripts/project.py` 仅保留为低层调试入口。
- 场景集合的单个表单新增和普通分布批量新增已归入 `backend.scenarios`；`core` 只读取场景文件并消费。
- `model train` 会导出 `graph/math_context.json`、`graph/samples/*.json`、`graph/dataset_profile.json`。
- `rules.tasks` 会从训练 samples 推断 `max_slots`、`count_bounds` 和 `param_bounds`。
- VAE reader 支持共享 context + 多 learning sample。
- VAE generation reader 支持只读 context graph。
- VAE typed message passing 已接入 `edge_index + edge_attr`。
- edge feature 已精简到必要关系强度。
- `generation` 已合并 model generate 和 decode，直接输出 scenario set。
- model generate 的中间 math graph 仅放在临时目录，decode 完立即清理。
- target-copy 作为管道连通性检查保留。
- generated graph 可解码为 disturbance graph，并直接回灌为 scenario set。
- 场景集和数据集消融分析由 Web 调用后端实时读取当前项目产物计算，不再提交静态报告产物。
- 求解和导出时刻表不再写 dataset 级完成标志；完成度由每个 case 的 `.sol`、`.sol.csv` 和 `adjusted_timetable.json` 推导。
