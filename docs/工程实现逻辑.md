# 工程实现逻辑

本文记录当前工程边界。实验流程说明看 `docs/exp.md`，断兼容升级说明看 `docs/version.md`，扰动图理论看 `docs/扰动图理论.md`。

## 分层原则

```text
frontend
  负责交互、任务状态、结构化数据展示和图形化分析

backend
  负责 API、项目目录、数据流、任务快照、Pueue 队列、日志、场景构造、批量流程和分析数据聚合

core
  负责原计划解析、MILP 建模、求解、扰动图转换、VAE 数学图转换等原子算法

VAE
  负责数学图训练和生成，不读取项目目录、YAML 场景、MILP 变量或 UI 状态
```

核心约束：

- `core` 不出现 workflow 包，不创建 project，不管理 dataset/model/scenario_set 生命周期。
- `backend` 是唯一流程编排层，所有耗时操作都通过 `backend.runner` 读取任务快照执行。
- `frontend` 不直接读写文件系统，只访问 `/api`。
- 不保留旧版 workflow CLI 和配置文件兼容入口。

## 当前关键文件

```text
main.py
  启动 FastAPI，检查 frontend/dist 是否存在

backend/app.py
  API 路由和静态前端分发

backend/service.py
  Web/API 服务门面，负责项目、任务、实体操作入口

backend/task_contracts.py
  任务参数默认值、校验、快照执行分发

backend/runner.py
  Pueue 任务统一入口：python -m backend.runner <input.json>

backend/workflow.py
  backend 数据流编排：prepare/build/solve/train/generation 等流程

backend/state.py
  project 状态扫描，供 Web 页面展示

backend/scenarios.py
  场景集合管理、手动场景写入、批量场景采样

backend/analysis/
  读取项目产物并计算前端展示所需结构化数据

core/base_context.py
  从时刻表和里程表构建 BaseContext

core/builder.py
  基于 BaseContext + ScenarioConfig 构建 MILP 模型

core/solver.py
  求解 LP 并读取 solution

core/vae_learning_graph.py
  扰动场景、扰动图和 VAE 数学图之间的原子转换

scripts/train_vae.py
scripts/generate_vae.py
scripts/evaluate_vae.py
  模型训练/生成/评估的低层算法脚本，由 backend workflow 调用或用于单点调试
```

## 数据流

```text
source timetable / mileage
  -> projects/<project>/context.json
  -> projects/<project>/scenario_sets/<scenario_set>/*.yml
  -> projects/<project>/datasets/<dataset>/cases/<case>/<case>.lp
  -> projects/<project>/datasets/<dataset>/cases/<case>/<case>.sol
  -> projects/<project>/datasets/<dataset>/cases/<case>/adjusted_timetable.json
  -> frontend visualization / backend analysis
```

模型流程：

```text
context.json + scenario_sets/<scenario_set>/
  -> backend/workflow.py 导出 model/<model>/graph/
  -> scripts/train_vae.py 训练 best_model.pt / last_model.pt
  -> scripts/generate_vae.py 生成临时数学图
  -> backend/workflow.py 解码成 scenario_sets/<target>/*.yml
```

中间生成图不作为正式项目产物保留。

## 任务系统

所有耗时行为都先写入任务快照：

```text
var/tasks/<project>/<timestamp>_<action>/input.json
```

Pueue 只执行统一命令：

```text
python -m backend.runner <input.json>
```

任务状态以 Pueue 为准；输入参数、项目 ID、action 和创建时间以 `input.json` 为准；日志由 Pueue/local log 读取。

## 产物约定

正式项目产物只保存在 `projects/<project>/`：

```text
context.json
scenario_sets/<scenario_set>/*.yml
datasets/<dataset>/cases/<case>/scenario.yml
datasets/<dataset>/cases/<case>/build.json
datasets/<dataset>/cases/<case>/<case>.lp
datasets/<dataset>/cases/<case>/solve.json
datasets/<dataset>/cases/<case>/<case>.sol
datasets/<dataset>/cases/<case>/<case>.sol.csv
datasets/<dataset>/cases/<case>/adjusted_timetable.json
model/<model>/graph/
model/<model>/best_model.pt
model/<model>/last_model.pt
model/<model>/training_config.json
model/<model>/training_summary.json
model/<model>/schema_summary.json
model/<model>/loss_history.jsonl
```

跨数据集、跨场景集的总结性分析产物不落盘；需要时由 backend analysis 按现有产物即时计算。

## 已移除的历史入口

以下入口不再属于当前架构：

- `core/workflow/*`
- `scripts/project.py`
- 依赖 `core.loader.load_config()` 的旧导入/导出脚本
- 直接导出/解码中间扰动图 JSON 的旧调试脚本
- `.runs` 旧任务记录
- `build.yml`、`solve.yml`、`train.yml` 等隐藏流程配置

旧版数据处理和升级规则见 `docs/version.md`。
